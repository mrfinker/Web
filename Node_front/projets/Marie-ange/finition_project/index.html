Technologie.

Python.

Django.

Django est un cadre de developpement web qui aide à construire et à maintenir des applications web de qualité.
Django aide à eliminer les taches repetitives rendant le processus de developpement une experience facile pour temps moindre.

Framework Python pour le web django

Django est un cadre web Python de haut niveau qui encourage le developpement rapide, le design propre et pragmatique.

Histoire de Django

Debut en 2003 par Adrian Holovaty et Simon Willison comme projet interne

Sortie en 2005 et nommer Django, du Guitariste Jazz Django Reinhardt

Django est maintenant un projet open source avec des contributeurs à travers le monde

Philosophie

Django vise à faire chaque élément de sa pile independante des autres.

Moins de code donc à son tour un developpement rapide et flexible.

Don't Repeat yourself : Tout ne doit etre developpé que dans un seul endroit au lieu de le repeter encore et encore.

Django conserve stritectement un design propre tout au long de son propre code et facilite le suivi des meilleurs pratiques de developpement web

Details

Django fournit un pont entre le modele de données et le moteur de base de données en prennant en charge un grand nombre de systemes de base de données

Django prend en charge les sites Web multilangues grace à son systeme d'internationalisation integré. Ainsi vous pouvez developper votre site web, qui prendrait en charge plusieurs langues

Django prend egalement en charge la base de données NoSQL

Django a un support integré pour Ajax, RSS, Caching et divers

Django fournit une belle interface utilisateur prete à l'emploi pour les activités admninistratives

Django est livré avec un serveur web leger pour faciliter le developpement et les tests des applications de bout en bout

Modele view controller and Template

Django prend en charge le modele MVC.

Nous parlons generalement du modele MVC sur les applications UI, comme le nom l'indique ce modele est basé sur trois composants :
Modele, Vue et le controleur.

Le modele MVT est legerement different de MVC, la principale difference entre les deux modeles est que Django lui-meme s'occupe de la partie controleur.

"Code logiciel qui controle les interactions entre le modele et la vue"

Le modele est un fichier HTML melangé avec le langage modele Django(DTL)
